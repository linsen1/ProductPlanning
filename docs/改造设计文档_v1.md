# ProjectAI 产品策划智能体 · 改造设计文档 v1

> 目标：让系统稳定产出"像《2026 康养校庆产品体系》那样厚、那样能落地"的产品策划方案，而不是现在这种结构齐全但内容单薄的稿子。
> 本文档为**设计稿（动代码前的最终确认件）**，确认后再分期改代码与知识库。

---

## 0. 改造目标与范围

**一句话目标**：把"结构对、内容空"的方案，升级为"结构对、内容厚、能落地、带竞品、可直接用"的方案。

**已确认的方向（前期问答结论）**
- 交付形态：完整可复用的生成系统（在现有 Streamlit + LangGraph + RAG 架构上升级，不推倒重来）。
- 产品形态：支持**产品体系方案**（多证书 / 多班型），同时保留**单产品方案**模式。
- 竞品分析：内置竞品对比模块，联网调研补数据。
- 知识库：覆盖优路全项目，在现有"简单版"知识库基础上**重构 + 补素材**。
- 输出格式：docx 主交付 + 可选 HTML 预览。
- 推进方式：先出本设计文档 → 确认 → 分期改造。

**优先工作线（已确认）**：① 知识库重构　③ 模板加厚+体系模式　④ 竞品挖深+固化　⑤ 格式规范+docx 修复。
② 表单→关联模板检索作为串联 ①③ 的底层机制，一并纳入。

## 1. 现状架构与根因诊断

### 1.1 现状链路

```
填表(render_form, 9必填+可选竞品/结构字段)
  → 第一段 LangGraph: build_query → 博查搜竞品(3轮query) → STRATEGY_PROMPT 生成 3个USP方向
  → 策划人选方向
  → 第二段 LangGraph: retrieve_kb(similarity_search, k=3) → PRODUCT_PLAN_PROMPT 生成9章方案
  → export_to_word / 下载md
```

知识库现状：`data/` 平铺，6 份范例案例（case_*，质量参差，1.8KB~8KB）+ 6 份 project_kb 策划映射模板（template_00 总映射 + 01~05 分赛道）。Chroma 向量库，chunk_size=600，TOP_K=3。

### 1.2 根因（为什么单薄）

| 编号 | 根因 | 证据 | 后果 |
|---|---|---|---|
| R1 | **知识库只有"怎么写"，没有"写什么"** | project_kb 模板全是元指令（如"课程体系建议写成基础→实操→题库"），缺技能课清单、变现话术、证书事实、优势金句 | AI 无米下锅，只能现编，输出空泛 |
| R2 | **表单→模板无硬映射** | `build_retrieval_query` 拼查询串后 `similarity_search(k=3)`，纯靠 embedding 相似度 | 康养产品可能召回工程案例片段，"找对应关联模板"靠运气 |
| R3 | **召回量太小** | TOP_K=3，chunk_size=600 | 进 prompt 的素材不到 1800 字，撑不起一份厚方案 |
| R4 | **模板章节不够厚 + 缺体系模式** | 9 章单产品模板，产品优势只是 3.1~3.6 短条；无实操技能课详解、无就业变现、无多证书简章 | 对不齐 PDF 的厚度 |
| R5 | **竞品分析不固化** | STRATEGY_PROMPT 的竞品拆解只用于选 USP，最终方案正文无竞品对比表；博查 query 偏泛 | 方案缺竞品视角，差异化论证无依据 |
| R6 | **docx 导出丢表格** | `export_to_word` 逐行解析 markdown，表格行被当普通段落 | 简章/课程说明两张表导出 Word 后散架 |

## 2. 工作线① 知识库重构（解决 R1/R3）

### 2.1 分层目录（从平铺改为 7 层）

```
data/
├── 00_structure/      结构模板层（怎么写）   ← 现有 template_00~05 迁入，保留
├── 01_cert_facts/     证书事实卡层（写什么·事实）每证书一份
├── 02_skill_modules/  实操技能课内容库（写什么·技能）按证书的技能课模块清单
├── 03_career_paths/   就业变现库（写什么·变现）线上/线下/全职三赛道
├── 04_copy_bank/      金句话术库（写什么·表达）Slogan/朋友圈/异议处理
├── 05_cases/          优秀成稿范例层           ← 现有 case_* 迁入
└── 06_compliance/     合规边界库               ← 从 template_00 抽离，全局复用
```

**分层逻辑**：00 管"骨架怎么搭"，01-04 管"血肉填什么"，05 管"成品长什么样"，06 管"红线在哪"。现在缺的就是 01-04 这四层血肉，这是补"厚度"的核心。

### 2.2 文件 front-matter 规范（供检索硬过滤）

每个 `.md` 顶部加 YAML：

```yaml
---
project_type: 短考类-营养健康      # 对应表单 PROJECT_TYPES，硬过滤用
cert: 公共营养师                   # 证书级（事实卡/技能课/变现库必填）
layer: cert_facts                  # structure|cert_facts|skill_modules|career_paths|copy_bank|case|compliance
priority: high                     # high 的内容优先进 prompt
applies_to: [短考类-营养健康, 短考类-技能认证]   # 可跨类型复用
---
```

### 2.3 各层内容规范（关键：定义"厚"的标准）

**01 证书事实卡**（每证书一份，对齐 PDF 招生简章那种颗粒）：发证机构、证书性质、级别（助理/初级/中级…）、报考条件、考试形式与时间、价格区间参考、适用岗位与场景、补贴/落户等可能场景（带边界）、合规红线。

**02 实操技能课内容库**（对齐 PDF"技能课1/2/3"，这是补厚度的重头）：按证书列出可拆解的技能模块清单。例如公共营养师 → 技能课1各人群膳食指南 / 技能课2日常营养实战配餐 / 技能课3慢病膳食干预；心理咨询师 → 职场成长板块 + 家庭实操（萨提亚）+ 10 大疗法清单。每个模块写"解决什么真实场景 + 学完能做什么"。

**03 就业变现库**（对齐 PDF"就业指导课"）：每证书的线上接单路径、线下兼职渠道、全职求职技能（简历模板要点 + 高频面试题 + 适配岗位），话术可直接套用，收益一律写成"取决于个人资源与当地市场"。

**04 金句话术库**：8 大优势的标准金句、Slogan 备选池、朋友圈三型文案模板、5 类异议的标准承接话术。让 AI 选词而非造词。

**06 合规边界库**：包过/押题/不过退/补贴必领/就业高薪等禁止表达 → 可替代表达对照表（从 template_00 抽出，全局注入每次生成）。

### 2.4 迁移与补素材计划

- **迁移**（不丢现有资产）：template_00~05 → `00_structure/`，case_* → `05_cases/`，template_00 的合规段 → `06_compliance/`，全部补 front-matter。
- **新建 01-04 四层素材**：以你那份《康养校庆产品体系》PDF 为蓝本，先把"营养师 / 心理咨询师 / 健康管理师"三证书的事实卡 + 技能课库 + 变现库做满（作为样板），其余证书/赛道按同结构留模板槽位，后续填充。
- **待你提供 / 我联网补**：各证书最新报考政策与价格（联网+你确认）、优路真实师资名单与数据（你提供，避免编造）。

## 3. 工作线② 表单→关联模板检索（解决 R2/R3）

### 3.1 现状问题

`retrieve_cases` 只做一次 `similarity_search(query, k=3)`，没有按 `project_type` 过滤，"找对应关联模板"完全依赖向量相似度，召回不可控、量也太小。

### 3.2 改造：硬过滤 + 分层软召回

检索分两段，结果合并注入 prompt：

**第一段·硬命中（不靠相似度，按表单字段必定取到）**
1. 按 `project_type` 命中 `00_structure/` 对应赛道模板（必进 prompt）。
2. 按勾选的 `cert[]`（证书多选）命中 `01_cert_facts/` 对应证书事实卡（每证书一张，必进）。
3. `06_compliance/` 合规库（全局必进）。

**第二段·分层软召回（embedding，带 metadata 过滤）**
在 `applies_to` 命中当前 `project_type` 的范围内，分层各取 N：
- `02_skill_modules`：取 3-4（技能课是厚度来源，多取）
- `03_career_paths`：取 2
- `04_copy_bank`：取 2
- `05_cases`：取 2（学写法，不抄）

**参数调整**：`TOP_K` 由 3 → 分层合计约 9-11；Chroma 检索改用 `similarity_search(..., filter={...})` 带 metadata 过滤；`chunk_size` 维持 600 但对 01/02 这类"整卡要完整"的层考虑整文件不切分（用 `priority:high` 标记整注入）。

### 3.3 表单字段变更（app.py / config.py）

| 字段 | 变更 | 说明 |
|---|---|---|
| 模式开关 | **新增** `plan_mode`：单产品 / 产品体系 | 决定走哪套输出模板 |
| 证书多选 | **新增** `certs[]` | 产品体系模式下多选，驱动多张事实卡 + 分证书简章 |
| project_type | 保留 | 仍是硬过滤主键 |
| 班型/价格梯度 | **增强** | 体系模式下支持多班型多价格录入 |
| 其余字段 | 保留 | 现有竞品/师资/课程结构可选字段不动 |

> 表单最终字段表在动代码前再与你逐字段确认（你之前选了"看过文件再定"，此处先给改造方向）。

## 4. 工作线③ 输出模板加厚 + 产品体系模式（解决 R4）

### 4.1 两套模板

- **单产品模板**：在现有 9 章基础上加厚（每章定下限、必含要素）。
- **产品体系模板（新增）**：对齐你那份 PDF，覆盖多证书 + 多班型 + 实操技能课 + 就业变现 + 三师服务 + 分证书简章。

### 4.2 产品体系模板章节骨架（对齐 PDF）

| # | 章节 | 现有 | 加厚/新增要点 | 字数/规模下限 |
|---|---|---|---|---|
| 0 | 主卖点 Slogan | 有 | 体系级主 Slogan + 各证书子 Slogan | 3 候选 + 推荐 |
| 1 | 产品背景 | 有 | 市场/用户/产品三层，加赛道趋势与人才需求 | ≥200 字 |
| 2 | 适合人群 | 有 | 改人物画像 + 情绪钩子（对齐 PDF"厌倦内卷…"），4 维拆分 | ≥5 类人群 |
| 3 | 产品优势 | 弱（3.1~3.6 短条） | **升级为 8 大优势块**：品牌实力 / 超高通过率 / 劲爆师资 / 智能题库 / 岗位实操技能课 / 就业指导变现 / 技术赋能(1:1机考+AI精灵) / 三师共管。每块=标题金句+展开论证+对应痛点 | 每块 ≥80 字 |
| 4 | **实操技能课详解（新增·核心）** | 无 | 每证书技能课逐模块拆（PDF 技能课1/2/3 那种），写清"真实场景+学完能做什么" | 每证书 ≥3 模块 |
| 5 | **就业指导/变现路径（新增）** | 无 | 线上接单 / 线下兼职 / 全职求职 三赛道，话术可套用，收益带边界 | 3 赛道齐 |
| 6 | 产品简章 | 弱（1 张表） | **按证书分别出简章表**，课程模块分"夯基/冲刺/从业"三层，配套资料分纸质邮寄+电子，产品服务细化（答疑双通道/协议畅学/含考务费等） | 每证书 1 表 |
| 7 | 师资介绍 | 有 | 保留，优先用真实师资，否则标"待确认" | — |
| 8 | 课程说明 | 有（表格） | 分阶段表 + 课时 + 上课形式 + 上线时间 | 修复表格导出 |
| 9 | 产介话术 | 有 | 一句话 / 朋友圈×3 / 1v1 / 社群，话术从 04_copy_bank 取 | 保留 |
| 10 | 产品搭建整体思路 | 有 | 体系逻辑 + 价格阶梯设计 + 竞品差异点 | ≥200 字 |
| 附 | **竞品对比模块（新增）** | 无 | 见工作线④，固化进正文 | 六维矩阵 |

### 4.3 加厚的强制手段（写进 prompt 的"质量纪律"）

1. **每章设字数/要素下限**，不达标必须补全，禁止一句话带过（尤其第 3/4/5 章）。
2. **第 4 章技能课必须逐模块展开**，模块来源于 `02_skill_modules` 召回内容；召回为空时按赛道通用技能模块兜底并标注。
3. **产品优势必须对应痛点**：每块优势后跟"解决的用户顾虑"。
4. **合规注入**：每次生成附带 `06_compliance` 红线，高风险表达自动转可替代表达。
5. 信息不足处统一标注"（参考，待策划人确认）"，不硬编数据。

## 5. 工作线④ 竞品分析挖深 + 固化（解决 R5）

### 5.1 现状

第一段 `bocha_search_node` 跑 3 轮 query（竞品+赛道大盘+差评），STRATEGY_PROMPT 做竞品拆解 → 但只服务"选 USP 方向"，**最终方案正文没有竞品对比表**，且 query 偏泛、弱点挖掘浅。

### 5.2 挖深：博查多轮检索升级

把搜索从"赛道泛搜"细到"竞品×维度"的矩阵式多轮：
- 竞品定价：`{竞品} {证书} 价格 班型 2026`
- 竞品卖点：`{竞品} {证书} 课程 优势 服务`
- 用户口碑/差评：`{竞品} {证书} 评价 差评 退费 投诉`（覆盖黑猫投诉/贴吧/知乎口径）
- 赛道大盘：`{证书} 头部机构 对比 2026`
- 政策/考情：`{证书} 报考条件 考试时间 2026 官方`

产品体系模式下，对每个证书各跑一组，结果按证书归并。

### 5.3 固化：竞品对比矩阵进正文

最终方案新增"竞品对比"模块，六维矩阵：

| 维度 | 竞品A | 竞品B | 竞品C | 我方(优路) |
|---|---|---|---|---|
| 主打卖点 | … | … | … | … |
| 价格区间 | … | … | … | … |
| 师资 | … | … | … | … |
| 服务体系 | … | … | … | … |
| 通过率话术 | … | … | … | … |
| 明显弱点/差评 | … | … | … | （我方对应的差异化补强）|

### 5.4 闭环：竞品弱点 → 我方优势

竞品的"明显弱点"列，自动映射成第 3 章"产品优势"里的差异化论证依据（如竞品差评"只讲理论"→ 我方优势"岗位实操技能课"）。竞品分析不再是孤立一段，而是贯穿优势论证的依据来源。

> 合规：竞品信息全部标注来源（博查搜索结果），未搜到的不臆造，写"搜索未覆盖"。

## 6. 工作线⑤ 格式规范 + docx 表格修复 + HTML 预览（解决 R6）

### 6.1 docx 导出重写（现有 export_to_word 的 bug）

现状逐行解析，markdown 表格被当普通段落 → 简章/课程说明/竞品矩阵三张表导出后散架。改造：
- 引入 markdown 表格解析：识别 `| … | … |` 与分隔行，生成真正的 Word `Table`，套表头底色/边框/品牌色。
- 加封面页（产品名 + 优路品牌色 + 日期）、自动目录、页眉页脚。
- 标题层级、加粗、列表、分隔线保留现有逻辑，补"表格 + 图注"。
- 用 docx 技能（python-docx）实现，动代码阶段读 `skills/docx/SKILL.md` 对齐规范。

### 6.2 HTML 预览版（可选交付）

输出一份单文件 HTML，贴近 PDF 排版观感：分证书卡片、优势块栅格、简章表格化、品牌色（优路朱红）。供运营快速预览/截图，不替代 docx。

### 6.3 《输出格式规范》清单（随系统沉淀一份）

- 每章字数下限与必含要素（见 4.2 表）。
- 表格统一规范：表头、对齐、单位、空值写法。
- 合规话术清单（禁止表达 → 可替代表达）。
- Slogan 规范：≤12 字、直白押韵、不加引号、3 候选。
- 话术规范：朋友圈 ≤150 字/条、社群 ≤100 字、口语化。

## 7. 分期实施计划

> 推进方式（已确认）：本设计文档定稿后再动代码。以下为定稿后的执行分期。

| 阶段 | 内容 | 主要改动文件 | 交付物 | 验收标准 |
|---|---|---|---|---|
| **P0 定稿** | 确认本文档 + 逐字段敲定表单 | 本文档 | 设计定稿 + 表单字段表 | 你签字确认 |
| **P1 知识库重构** | 建 7 层目录、迁移现有、补营养/心理/健管三证书的事实卡+技能课+变现库样板、加 front-matter | `data/**`、新增样板 md | 可检索的分层知识库 | 三证书素材齐，重建索引通过 |
| **P2 检索升级** | 硬过滤+分层软召回、metadata filter、TOP_K 分层、表单加模式/证书多选 | `app.py`、`config.py`、`prompts.py` 的 build_query | 检索按类型精准命中 | 康养产品稳定召回康养模板+对应证书卡 |
| **P3 模板加厚+体系模式** | 新增产品体系 prompt、8大优势、技能课/变现两章、分证书简章、质量纪律 | `prompts.py` | 两套模板 | 用健管示例生成，厚度对齐 PDF |
| **P4 竞品挖深+固化** | 博查多轮升级、竞品六维矩阵进正文、弱点→优势闭环 | `app.py`(bocha节点)、`prompts.py` | 方案含竞品对比表 | 矩阵有数据有来源 |
| **P5 格式交付** | docx 表格修复+封面目录、HTML 预览、格式规范文档 | `app.py`(export)、新增 html 导出 | docx 表格正常 + HTML | 表格不散架，预览可用 |

## 8. 风险与合规

- **不编造数据**：通过率、价格、师资、政策一律走"你提供 / 联网核实 / 标待确认"，禁止模型臆造。
- **合规红线**：包过、押题命中、不过退、补贴必领、就业高薪、证书效力等全部走可替代表达（06_compliance 全局注入）。
- **知识库维护**：front-matter 必须规范，否则硬过滤失效；新增证书按样板结构补，避免层级混乱。
- **成本**：TOP_K 提高 + 多轮博查会增加 token 与搜索调用，需在效果与成本间取平衡（可配置）。
